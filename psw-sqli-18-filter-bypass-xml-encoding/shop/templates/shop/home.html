Lab briefing: each product has a client-side Check stock button that posts an XML body to POST /product/stock:

<stockCheck><productId>1</productId><storeId>1</storeId></stockCheck>

The server rejects (HTTP 403) any request whose raw body contains SELECT, UNION, FROM, or WHERE. The blocklist check runs on the bytes before the XML parser does. Because the XML parser decodes numeric character references (&#x55; → U), encoding even one character of a keyword smuggles it past the filter; by the time the productId reaches the SQL string the keyword is back. The first column of the stock query is reflected verbatim in the response body — in-band UNION extraction of the administrator password fits in a single request. Then sign in to claim the flag.
