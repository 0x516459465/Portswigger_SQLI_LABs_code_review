Lab briefing: the site looks up the TrackingId cookie against a tracked_users table. The result is never reflected and any SQL error is swallowed - successful and failing payloads produce identical response bodies. The only observable server-side effect of a payload is wall-clock time. To solve, exploit the injection to keep the cookie lookup running for at least {{ threshold }} seconds. The backend is PostgreSQL; pg_sleep() is the intended delay primitive, e.g. TrackingId=x'||pg_sleep({{ threshold }})::text||'.
